# Codexa

> **Auto-generate structured markdown context files so AI coding agents (and humans) can navigate your codebase at a glance.**

Codexa walks your codebase and writes a `CODEXA.md` file into every directory containing Python source files. Each file is a self-contained knowledge document — powered by AST analysis and an LLM backend — covering module purpose, key symbols, dependency graphs, non-obvious patterns, and tribal knowledge hints. The result is a persistent, human-readable context layer that dramatically reduces the number of tool calls needed to understand a large or unfamiliar codebase.

---

## Quick Start

```bash
# Install
pip install codexa

# Generate CODEXA.md files for your project (uses OPENAI_API_KEY env var)
codexa generate ./my-project

# Preview what would be written without touching the filesystem
codexa preview ./my-project/src/core

# Remove all generated CODEXA.md files
codexa clean ./my-project
```

That's it. After running `generate`, every Python-containing directory will have a `CODEXA.md` ready for you or your AI agent to read.

---

## Features

- **AST-powered analysis** — Extracts top-level functions, classes, imports, and docstrings from Python source files using the standard library `ast` module for accurate, dependency-free structural metadata.
- **LLM summarization** — Calls any OpenAI-compatible endpoint to generate a plain-English module overview, flag non-obvious patterns, and surface tribal knowledge hints that don't live in the code itself.
- **Per-directory `CODEXA.md` generation** — Renders rich markdown files from a customizable Jinja2 template with sections for overview, key symbols, dependencies, caveats, and subdirectory navigation links.
- **Incremental regeneration** — Embeds a content hash in every generated file and skips unchanged directories on subsequent runs, keeping re-generation fast on large codebases.
- **Flexible configuration** — A `.codexa.toml` file controls the model, API endpoint, token budget, recursion depth, and ignore patterns (with `.gitignore`-style semantics via `pathspec`).

## Usage Examples

### Generate context files for an entire repo

```bash
export OPENAI_API_KEY="sk-..."
codexa generate . --depth 4
```

### Use a local or alternative OpenAI-compatible endpoint

```toml
# .codexa.toml
[codexa]
model      = "llama-3-8b-instruct"
base_url   = "http://localhost:11434/v1"
api_key    = "ollama"
max_tokens = 512
max_depth  = 3
ignore     = [".git", "__pycache__", "node_modules", "*.pyc", "dist"]
```

```bash
codexa generate ./my-monorepo
```

### Preview without writing

```bash
codexa preview ./my-monorepo/services/auth
```

Outputs the rendered markdown directly to the terminal — useful for quick inspection or CI checks.

### Force regeneration of all files (ignore hashes)

```bash
codexa generate . --force
```

### Clean up all generated files

```bash
codexa clean . --dry-run   # See what would be removed
codexa clean .             # Actually remove them
```

### Example generated `CODEXA.md` (excerpt)

```markdown
<!-- codexa-hash: a3f9c2e1... -->
# CODEXA: auth

> Generated by Codexa on 2024-11-01 | Directory: `services/auth`

## Overview
This module implements JWT-based authentication. Token validation is
handled by `verify_token()` which checks both signature and expiry.
Note: refresh tokens are stored in Redis, not the database.

## Key Symbols
- `verify_token(token: str) -> Claims`
- `AuthMiddleware` — ASGI middleware, applied at the app layer
- `TokenExpiredError`

## Non-Obvious Patterns
- Clock skew tolerance is set to 30 s; do not tighten without updating
  the mobile client.
```

## Configuration

Create a `.codexa.toml` in your project root. All fields are optional — Codexa ships with sensible defaults.

```toml
[codexa]
# LLM backend
model     = "gpt-4o-mini"          # Any OpenAI-compatible model name
api_key   = "sk-..."               # Overridden by OPENAI_API_KEY env var
base_url  = "https://api.openai.com/v1"  # Overridden by OPENAI_BASE_URL env var

# Analysis limits
max_tokens = 1024                  # Max tokens per summarization request
max_depth  = 5                     # Max directory recursion depth

# Ignore patterns (.gitignore semantics, powered by pathspec)
ignore = [
  ".git",
  "__pycache__",
  "node_modules",
  "*.pyc",
  "dist",
  ".venv",
]

# Custom Jinja2 template (leave empty to use the built-in template)
template = ""                      # e.g. "./my_template.md.j2"
```

### Environment Variable Overrides

| Variable | Overrides |
|---|---|
| `OPENAI_API_KEY` | `api_key` in config |
| `OPENAI_BASE_URL` | `base_url` in config |
